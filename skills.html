Skills

In web development, there are three fundamental skills used to make a fascinating website. HTML (Hyper Hypertext Markup Language) as a content layer (where the content of the page lives and gives the structure), CSS (Cascading Style Sheets) as a presentation layer (where styles are added to state how HTML content is presented), and ECMAScript also known as JavaScript as a behaviour layer (where we can change how the page behaves or interact with users). This page explains what CSS and ECMAScript are with showing example pages.

CSS

CSS stands for Cascading Style Sheets. It acts as a presentation layer on a website which means it adds styles to the HTML and changes how the HTML will be presented to users. There are three ways of adding CSS to an HTML file, inline CSS (way of adding CSS to specific tags using style attribute), internal CSS (way of adding CSS inside <head> tag using <style> tag), and external CSS (way of adding CSS by linking a external CSS file to an HTML file). The best way of adding CSS is external CSS since the CSS is separated from the HTML file, whereas inline CSS and internal CSS live inside the HTML file which makes the HTML cluttered and makes the size of the HTML file big. External CSS makes it easier to maintain the HTML file and the CSS file since it is separated and a single CSS file can be linked to multiple webpages, hence efficient.

ECMAScript

ECMAScript stands for European Computer Manufacturer's Association. ECMAScript is also known as JavaScript and it is called interchangeably, but strictly speaking, ECMAScript is the standard and JavaScript is the language (created by Brendan Eich in 10 days) which implements ECMAScript standard. The current latest version of ECMAScript is ECMAScript 2020, but ECMAScript 2017 is the current widely-supported standard. ECMAScript (JavaScript) acts as a behaviour layer on a website which means it changes how the page behaves or interacts with users i.e. it adds interactivity. Same as CSS it is better to keep JavaScript separate from the HTML file for efficiency, maintainability and to make the website work even if the user can't load or run the JavaScript file. JavaScript file is added to the last part of the body (before the </body> tag) using <script> tag. The reason for adding it to the last part of the body is because when a browser loads a JavaScript file it stops to load other files. This will make the website look slow to load. Another reason is that JavaScript file on a website requires an HTML file, therefore it is better to load the HTML file before the Javascript file.

The section below demonstrates the power of ECMAScript (JavaScript) by using a website for ordering coffee.

Coffee Ordering Website

This is a website for ordering coffee. You can order coffee by choosing a drink with any other options. Keep an eye on all interactions with each element. It will update the user's order concurrently. This is done by using event listeners. Also, when the "Add to favourites" button is clicked, it will record the current order and by clicking on the "Order favourite" button it will automatically fill out the form. This is done by using local storage. The feature that when you order 10 drinks the 10th drink will be free is also accomplished by using local storage. The user's favourite drink and the number of drinks purchased will not be erased even if the window or tab is closed.
